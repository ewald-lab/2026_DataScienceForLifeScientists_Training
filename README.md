# Data Science for Life Scientists: Cell Painting Project

This repository contains the practical materials for a project-based training course in data science for life scientists. Students will work in groups to explore Cell Painting data, connect image-derived morphology profiles to chemical annotations, and build predictive models for biological activity.

## Biological Question

The central project question is:

**Can we use Cell Painting morphology profiles to predict chemical cytotoxicity or mode of action?**

More specifically, students will ask whether high-content microscopy images contain enough phenotypic information to classify chemicals as active or inactive for selected biological endpoints. Depending on the group project, the endpoint may be an in vitro cytotoxicity label, a cell-based ToxCast assay, or another biological activity label that can be linked to the compounds.

This is a realistic problem in chemical biology and toxicology. Cell Painting assays measure many cellular structures in parallel, creating rich morphology profiles for each compound. Machine learning models can then test whether those profiles contain predictive signals for toxicity, pathway activity, or mechanism-related phenotypes.

## Data Context

The course uses Cell Painting and ToxCast-style data tables stored in `inputs/`.

Cell Painting is a microscopy assay where cells are treated with compounds, stained across multiple biological channels, imaged, and converted into quantitative morphology features. The raw images are not the main training data for most models in this course; instead, students will mostly use processed feature tables derived from those images.

Key files include:

- `inputs/load_data.csv`: image manifest with S3 paths for Cell Painting channels, including AGP, ER, mitochondria, DNA, RNA, and brightfield images. This is used to inspect example microscopy fields.
- `inputs/BR00127145.parquet` to `inputs/BR00127149.parquet`: [Exemple data for cell painting processing protocol] plate-level CellProfiler-style morphology profiles used in the processing demonstration. Each plate has one row per well and thousands of morphology features.
- `inputs/compound_well_map.csv`: [Exemple data for cell painting processing protocol] links plates and wells to compound identifiers.
- `inputs/compound_info.csv`: [Exemple data for cell painting processing protocol] compound chemistry metadata such as InChIKey, InChI, and SMILES.
- `inputs/perturbation_controls.csv`: [Exemple data for cell painting processing protocol] treatment annotations, including controls such as DMSO and positive-control compounds.
- `inputs/level6_axiom_selected_features.parquet`: [Actual dataset you will analyse] final selected Cell Painting profiles used for downstream modelling. In this repository, this table contains 1,086 compound-level profiles and 4,439 columns.
- `inputs/v5_oasis_03Sept2024_simple.csv`: [Actual dataset you will analyse] compound metadata used to connect Cell Painting compound names to `OASIS_ID` identifiers.
- `inputs/toxcast_cytotox_binary.parquet`: [Actual dataset you will analyse] binary cytotoxicity labels. A value of `1` means active and `0` means inactive for a selected endpoint.
- `inputs/toxcast_cellbased_binary.parquet` and `inputs/toxcast_cellfree_binary.parquet`: [Actual dataset you will analyse] additional binary biological activity endpoints that can be used for mode-of-action or assay-specific prediction tasks.

The main modelling workflow is:

1. Start from processed Cell Painting morphology profiles.
2. Map compounds to shared identifiers.
3. Merge morphology features with ToxCast labels.
4. Choose one prediction endpoint.
5. Train, evaluate, and interpret machine learning or deep learning models.

## Repository Structure

- `analysis/01_data_collection_preprocessing/`
  - `01_plot_images.ipynb`: inspect example Cell Painting image channels.
  - `02_cell_painting_pipeline.ipynb`: demonstrate how raw image-derived profiles are processed into normalized and aggregated morphology signatures.
  - `03_data_handling_visualising.ipynb`: understand the processed modelling dataset, merge tables, choose cytotoxicity endpoints, and visualize morphology patterns.
- `analysis/02_simple_ml/`
  - `01_ml_protocol.ipynb`: build a classical machine learning protocol for endpoint prediction.
- `analysis/03_deep_learning/`
  - `01_dl_protocol.ipynb`: build a simple PyTorch deep learning protocol.
- `analysis/outputs/`: intermediate outputs and figures generated by notebooks.
- `inputs/`: data tables and image manifests.
- `requirements.txt`: Python package requirements.

## Course Schedule

### Day 1: Group Formation and Project Planning

- Form project groups.
- Refine your protocol to answer the biological question.
- Define the project plan:
  - Which input features will you use?
  - Which endpoint will you predict?
  - What data exploration wou will implement?
  - How you will develop your machine learning pipeline?
  - What evaluation metric is biologically meaningful?
- Present project plans to mentors.
- Receive mentor feedback and update the plan before starting to explore the proposed analysis inthe notebooks.

### Day 2: Cell Painting Data and Preprocessing

Open `analysis/01_data_collection_preprocessing/`.

Start with `01_plot_images.ipynb` and move quickly through it to see what the example Cell Painting image data look like. The goal is to understand the channels and the structure of microscopy fields, not to spend the day training models on raw images.

Then move to `02_cell_painting_pipeline.ipynb` to see how Cell Painting images are converted into morphology profiles. This notebook uses an example dataset to illustrate the processing workflow. It is not the final dataset used for machine learning model training.

Spend most of the day on `03_data_handling_visualising.ipynb`. This is the key notebook for understanding the processed dataset used later in the course. Students should learn what one row represents, how Cell Painting features are merged with compound and ToxCast metadata, how binary endpoints are selected, and whether active compounds show visible morphology patterns.

### Day 3: Classical Machine Learning

Open `analysis/02_simple_ml/01_ml_protocol.ipynb`.

Develop a machine learning protocol to predict an in vitro cytotoxicity endpoint or a biological mode-of-action-related endpoint from Cell Painting features. The focus is on building a reproducible modelling workflow:

- select the target endpoint;
- define feature and label tables;
- split data into training and test sets;
- prevent data leakage during preprocessing;
- train baseline and supervised models;
- evaluate performance with appropriate metrics;
- interpret which morphology features or compounds drive predictions.

### Day 4: Deep Learning with PyTorch

Open `analysis/03_deep_learning/01_dl_protocol.ipynb`.

Develop a deep learning protocol using PyTorch. Students will adapt the same biological prediction problem to a neural network workflow:

- convert Cell Painting feature tables into tensors;
- define training, validation, and test loaders;
- build a feed-forward neural network;
- train with binary classification loss;
- monitor overfitting and class imbalance;
- compare deep learning performance with the classical machine learning approach.

### Day 5: Project Wrap-Up and Flash Talks

Prepare a 5-minute flash talk describing the group project.

The talk should cover:

- the biological question and selected endpoint;
- the data used and how it was prepared;
- the modelling approach;
- the performance result;
- what the group achieved and what they would try next with more time.

Keep the talk short and visual.
